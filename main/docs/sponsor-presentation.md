# PressPass Platform

## Цифрові журналістські посвідчення нового покоління

**Довіра. Перевірка. Безпека. У смартфоні.**

Платформа для видачі, адміністрування та миттєвої перевірки електронних журналістських посвідчень.

---

# Проблема

## Паперові й пластикові посвідчення більше не відповідають реальності

- Їх складно швидко перевірити.
- Підробки важко відрізнити від справжніх.
- Відкликане посвідчення може продовжувати використовуватись.
- Редакції витрачають час на ручне адміністрування.
- Журналісту потрібно носити фізичний документ, який можна втратити або пошкодити.

**У час, коли інформаційна довіра є критичною, журналістське посвідчення має бути цифровим, перевірюваним і захищеним.**

---

# Рішення

## PressPass — посвідчення, яке можна перевірити за секунди

PressPass надає три ключові інтерфейси:

1. **Журналіст** бачить посвідчення у PWA-застосунку на смартфоні.
2. **Адміністратор або редакція** видає, оновлює, блокує та керує посвідченнями.
3. **Громадськість, організації, охорона подій** перевіряють дійсність посвідчення через QR.

---

# Як це працює

## Простий сценарій

### Журналіст

- Реєструється.
- Заповнює анкету.
- Отримує цифрове посвідчення.
- Показує QR-код зі смартфона.

### Перевіряючий

- Сканує QR-код.
- Відкриває сторінку перевірки.
- Бачить актуальний статус посвідчення.

---

# Для редакцій та організацій

## Єдиний центр керування посвідченнями

Адміністратор може:

- створювати профілі журналістів;
- додавати журналістів до редакцій;
- видавати посвідчення;
- продовжувати строк дії;
- блокувати недійсні посвідчення;
- керувати дизайном картки;
- перевіряти актуальність даних.

**Менше ручної роботи — більше контролю та прозорості.**

---

# Безпека

## QR без персональних даних

QR-код містить лише посилання для перевірки, а не приватні дані журналіста.

Ключові переваги:

- серверна перевірка статусу;
- короткоживучий динамічний QR;
- захист від використання старих скриншотів;
- можливість швидко заблокувати посвідчення;
- публічна сторінка показує тільки потрібні для перевірки дані.

---

# Технологічна основа

## Сучасна архітектура, готова до масштабування

```text
PWA (Next.js) → REST API (NestJS) → Prisma ORM → PostgreSQL
```

Технології:

- **Frontend:** Next.js, React, Tailwind CSS, PWA.
- **Backend:** NestJS, Prisma ORM, PostgreSQL, JWT, REST API.
- **Якість:** TypeScript strict, ESLint, Prettier, CI.
- **Безпека:** HTTPS, JWT, Argon2, UUIDv7, rate limiting, Helmet, CORS.

---

# PWA: застосунок без App Store

## Посвідчення працює у смартфоні

PressPass запускається як PWA:

- відкривається у браузері;
- встановлюється на головний екран;
- не потребує окремого Android/iOS застосунку на старті;
- може працювати з офлайн-кешем останнього посвідчення;
- готовий до майбутніх інтеграцій із Flutter, Google Wallet, Apple Wallet і push-повідомленнями.

# Деплой та експлуатація

## Запуск на звичайному VPS

Платформа має сценарій розгортання одним скриптом:

```bash
sudo bash deploy/install.sh
```

Він автоматизує:

- встановлення Node.js, PostgreSQL, Nginx, PM2;
- налаштування домену та HTTPS;
- створення конфігурації;
- міграції бази даних;
- production-збірку;
- запуск сервісів.
