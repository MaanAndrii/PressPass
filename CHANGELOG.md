# Changelog

All notable changes to this project are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.1] — 2026-07-17

### Fixed

- Інсталяційний візард (`deploy/install.sh`) більше не обривається одразу після
  вводу останніх облікових даних на VPS без попередньо встановленого `openssl`:
  генерація секретів на етапі запитань тепер працює через `openssl` **або**
  `/dev/urandom` (coreutils), бо сам `openssl` доустановлюється лише пізніше.

### Security

- Завершено owner-key фундамент Етапу 1: версійні AES-256-GCM envelopes, випадкові DEK/KEK,
  Argon2id wrapping/rewrapping, окремі Admin/Editorial/System key slots та дві RSA-OAEP offline
  Superadmin recovery authorities.
- Email і Google subject шукаються через незалежні HMAC blind indexes; зміст користувачів,
  журналістів, редакцій, посвідчень, налаштуван і шаблонів зберігається лише у versioned encrypted
  payloads.
- Приватні фото й логотипи переведено на magic-byte-validated encrypted storage з авторизованою
  видачею; прямий static `/uploads` вимкнено в API та Nginx.
- Додано короткоживучі in-memory API unlock sessions, окрему encryption passphrase для Google і
  адміністраторів, resumable security backfill та обовʼязковий plaintext/key-slot verifier.

## [0.17.0] — 2026-07-15

### Changed

- **Розділення PWA та адмінки.** Встановлюваний застосунок (PWA) тепер суто журналістський — реєстрація, вхід і перегляд посвідчень; адмінка `/admin` більше **не входить** до PWA: service worker її не кешує й не реєструється на її сторінках (кеш піднято до `presspass-v4`). Адмінпанель можна винести на **окремий домен для ПК** змінною `PP_ADMIN_DOMAIN` при встановленні — тоді Nginx блокує `/admin` на основному домені, а на адмін-домені вимикає PWA (`sw.js`/`manifest` → `404`). Без цієї змінної поведінка не змінюється (один домен, `/admin` за шляхом). Деталі — у `docs/deploy.md`.

## [0.16.0] — 2026-07-15

### Added

- **Зміна пароля** для будь-якого адміністратора (і журналіста): розділ «Обліковий запис» в адмінці + `PUT /me/password` (перевіряє поточний пароль).
- **Редагування посад** у довіднику (кнопка «Редагувати»).

### Changed

- Модальні вікна в адмінці більше **не закриваються кліком повз них** — лише кнопкою «✕» (щоб не втратити напівзаповнену форму).
- **Усі видалення** в системі відбуваються через підтвердження.

### Fixed

- У конструкторі дизайну маркер виділення тепер **повертається разом із текстовим полем** (раніше рамка й держак лишалися горизонтальними при повороті на 90°).
- **Заборонено видаляти посаду**, яку вже використовують видані посвідчення (`409`).
- **Заборонено видаляти редакцію**, якщо в ній є журналісти (`409`) — спершу приберіть їх.

## [0.15.0] — 2026-07-14

### Fixed

- **Приватність даних журналістів.** Редакційний адміністратор більше **не бачить усіх журналістів системи** — лише тих, хто належить до його редакції. Самозареєстровані та чужі журналісти залишаються прихованими (`GET /admin/journalists` фільтрує за членством; системний адміністратор і далі бачить усіх).

### Added

- **Унікальний ID журналіста** (`JR-XXXXXX`, без плутаних символів). У кабінеті журналіста він показується великим шрифтом із кнопкою **«Скопіювати ID»** та підказкою передати його адміністратору медіа; там же — перелік редакцій, до яких журналіста додано (слугує підтвердженням).
- **Додавання журналіста за ID.** Редакційний адміністратор (або системний) додає наявного журналіста до медіа за його публічним ID: `POST /admin/journalists/attach { publicId }`. Ввід нормалізується (регістр, префікс), повторне додавання — ідемпотентне, невідомий ID → `404`.
- **Членство журналіст ↔ редакція** (багато-до-багатьох, таблиця `editorial_memberships`). Видача посвідчення автоматично додає членство; редакційний адмін може **прибрати журналіста зі своєї редакції** (не видаляючи акаунт): `DELETE /admin/journalists/:id/membership`.

### Migration

- Існуючим журналістам згенеровано публічні ID; членство **зворотно заповнено** з наявних посвідчень, тож редакційні адміністратори не втрачають доступу до тих, кому вже видали посвідчення.

## [0.14.0] — 2026-07-14

### Added

- **Шаблон нумерації посвідчень для кожної редакції.** У картці редакції адміністратор задає **унікальний префікс** і складає **шаблон номера** з токенів `{prefix}`, `{year}`, `{YY}`, `{seq}`, `{seq:N}` (з нулями), `{mediaId}` — із живим прев'ю. Лічильник **скидається щороку** і рахується окремо для кожної редакції, тож номери різних редакцій не перетинаються (унікальність префікса перевіряється, а глобально-унікальна колонка номера — останній запобіжник). Без префікса лишається спільна нумерація `PP-рік-№`.
- **Кілька посвідчень в одного журналіста.** Якщо журналіст має посвідчення від різних редакцій, у застосунку зверху зʼявляється **перемикач** (чіп на кожне посвідчення зі статусом), а кнопка «Зробити основним» задає основне. За замовчуванням відкривається основне, інакше — чинне найновіше. `GET /cards`, `PUT /card/primary`; `GET /card/qr?cardId=…` видає QR для вибраного посвідчення.
- **Поворот текстових полів на 90°** у конструкторі дизайну (кроком 90°, кнопки ↺/↻ у панелі властивостей).

### Changed

- Свайп на екрані посвідчення й далі перемикає мову (укр./англ.); між посвідченнями перемикаються чіпи зверху — щоб не перевантажувати жести.

## [0.13.0] — 2026-07-14

### Added

- **Дизайн посвідчення — окремо для кожної редакції.** Конструктор перенесено з окремої вкладки у розділ **«Редакції»**: у кожної редакції своя кнопка «Дизайн» і власний макет. Картка журналіста рендериться за дизайном редакції-емітента; якщо власного немає — успадковується системний стандартний. `GET /card-template?editorialId=…`, `PUT/POST /admin/card-template?editorialId=…` (редакційний адмін — лише своя редакція).
- **Чекбокс «Член НСЖУ»** у картці журналіста (створення/редагування) + бейдж у картці перегляду.
- **Логотип НСЖУ**: завантаження/видалення в розділі **«Налаштування»**; у конструкторі дизайну доступне нове джерело даних для зображення — **«Логотип НСЖУ»**, який відображається **лише для членів НСЖУ**. Публічний ендпойнт `GET /branding` віддає шлях логотипа для рендерингу картки; на сторінці перевірки для членів показується позначка НСЖУ.

### Changed

- Якщо посвідчення **анульоване** (заблоковане) або протерміноване, застосунок журналіста показує це **одразу при запуску** — помітним банером над карткою, а не лише під час перевірки QR.

## [0.12.0] — 2026-07-14

### Added

- **Drag-and-drop конструктор посвідчення** (новий режим макета `absolute`): кожне поле — вільно розташований елемент із власними координатами, розміром і стилем.
  - **Фонова сітка** (крок налаштовується, стандарт 10 px) допомагає вирівнювати; при переміщенні елемент **прилипає до вузлів сітки** (`Math.round(x / gridSize) * gridSize`) та до центру/країв полотна з **направляючими лініями**.
  - **Типи полів**: `text`, `image`, `qr`, `date`. Текст/зображення прив'язуються до реальних даних картки (ПІБ, посада, редакція, ідентифікатор медіа, номер, дата, фото, логотип, QR) або є власними.
  - **Панель властивостей** — динамічна за типом: джерело даних, текст, координати `x/y/width/height`, розмір і колір шрифту, жирний/курсив/великі літери, вирівнювання, фон поля; дублювання, порядок шарів, видалення.
  - **Режим перегляду** ховає сітку й направляючі — видно чисту картку.
  - **Експорт/імпорт JSON** макета.
- API `card-template` розширено полями `layoutMode` (`flow`/`absolute`), `gridSize` та `elements[]` з повною санітизацією (координати обрізаються в межах полотна, невідомі типи/джерела відкидаються, максимум 40 елементів).

### Changed

- Раніше збережені шаблони й уже видані посвідчення не зачіпаються: стандартний режим лишається `flow` (адаптивний), а на `absolute` картка переходить після збереження в новому конструкторі.

## [0.11.0] — 2026-07-14

- Вимкнено автозаповнення браузером логіну/пароля у формах створення (журналіст, адміністратор).

## [0.10.1] — 2026-07-14

### Fixed

- Видача посвідчення після видалення інших посвідчень більше не падає з «A card with this number already exists»: номер тепер обчислюється з найбільшого наявного за рік (а не з кількості рядків) і пропускає зайняті номери.

### Changed

- Розділ «Адміністратори»: системний адміністратор може створювати й видаляти **системних** адміністраторів (не лише редакційних). Захист від блокування: не можна видалити власний обліковий запис або останнього системного адміністратора.

## [0.10.0] — 2026-07-13

### Added

- **Два рівні адміністрування**: системний адміністратор (бачить усю систему, керує адміністраторами) і редакційний адміністратор, прив'язаний до однієї редакції. Розділ «Адміністратори» (`/admin/admins`) — створення/видалення редакційних адміністраторів. Редакційний адмін може лише: видавати/видаляти посвідчення від своєї редакції, створювати журналістів, редагувати профіль своєї редакції.
- **Видалення посвідчень** — кнопка та `DELETE /admin/cards/:id` (редакційний адмін — лише свої).
- Профіль редакції отримав **назви для відображення в посвідченні** (укр./англ.) — на картці показується дисплей-назва, а не повна юридична.

### Changed

- Посвідчення видаються **виключно від доданої редакції** (`editorialId` обов'язковий при видачі). **Посаду** для посвідчення задає редакція під час видачі (зберігається на самому посвідченні, укр./англ.) — журналіст її більше не вказує; поля «Посада»/«Редакція» прибрано з форм журналіста та анкети.
- Видалення журналіста тепер доступне лише системному адміністратору. Список адміністраторів переїхав із «Налаштувань» у розділ «Адміністратори».

### Database

- `Role` отримав значення `EDITORIAL_ADMIN`; `users.editorial_id` (прив'язка редакційного адміна); `editorials.display_name_uk`/`display_name_en`; `cards.position`/`position_en`. Міграція `20260713140000_admin_tiers_card_position`.

## [0.9.0] — 2026-07-13

### Added

- Розділ адмінки **«Редакції»** — довідник компаній-емітентів. Поля: назва, код ЄДРПОУ, офіційний сайт (реєстр посвідчень), логотип, директор, електронна адреса, фізична адреса, контактний телефон. CRUD + завантаження логотипа (`/admin/editorials`).
- При **видачі посвідчення** можна вибрати редакцію зі списку (`editorialId`). Назва й **логотип** обраної редакції підтягуються на картку з реєстру компаній (замість текстового поля журналіста та типового логотипа шаблону).
- Сторінка публічної перевірки показує компанію-емітента: назву, логотип і посилання на реєстр посвідчень.

### Database

- Нова таблиця `editorials`; у `cards` додано `editorial_id` (`ON DELETE SET NULL` — видані посвідчення зберігаються після видалення редакції). Міграція `20260713120000_editorials`.

## [0.8.0] — 2026-07-12

### Changed

- Посвідчення повністю перебудовано під екран смартфона: картка має фіксовані пропорції (ширина × висота) і рівномірно масштабується, щоб уміститися на будь-якому екрані, зберігаючи вигляд. Хедер і футер мають фіксовану висоту.
- Основну частину поділено на три зони: **верхня** — фото + ПІБ і посада; **середня** — логотип + редакція; **нижня** — номер і дата дії. Для кожної зони в дизайні задається вільне розміщення (по горизонталі: зліва/центр/справа; по вертикалі: зверху/центр/знизу) та власний інтерліняж.
- Дизайн: додано регулювання висоти картки, висоти хедера і футера; прибрано окремі перемикачі розташування логотипа, «Дійсне до» і номера (тепер це керується позиціюванням відповідних зон).

## [0.7.1] — 2026-07-12

### Added

- Дизайн картки: вибір сімейства шрифтів (системний / без засічок / із засічками / моноширинний / заокруглений) та інтерліняжу (міжрядковий інтервал); окремий розмір шрифту для номера посвідчення; логотип тепер можна розміщувати ще й під фотографією; максимальний розмір логотипа збільшено до 160 px.
- Налаштування: розділ «Адміністратори системи» зі списком усіх адміністраторських облікових записів (пошта, дата створення, статус підтвердження) — `GET /admin/settings/admins`.
